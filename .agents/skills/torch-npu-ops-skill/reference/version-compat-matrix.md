## CANN / PyTorch / torch_npu 版本配套关系（示例骨架）

> 说明：本文件提供一个**示例矩阵结构**与典型组合，用于 Skill 说明“官方推荐组合”与“版本不匹配风险”。具体版本号需根据实际项目所用文档进行调整和补充。

### 1. 示例版本矩阵（仅示意）

| 组合编号 | CANN 版本（Ascend Toolkit） | PyTorch 版本 | torch_npu 版本 | 备注 |
|---------|-----------------------------|--------------|----------------|------|
| A1 | 7.0.x | 2.1.0 | 2.1.0 | 假设为某一代官方推荐组合，支持主流训练/推理算子 |
| A2 | 7.0.x | 2.3.0 | 2.3.0 | 更高版本 PyTorch，适配更多动态图与编译特性 |
| B1 | 8.0.RC1 | 2.7.1 | 2.7.1 | 新一代组合，可能支持更多算子与更优性能 |

> 注意：真实项目中请以官方 README / Release Note 为准，这里只展示矩阵组织方式。

### 2. Skill 使用方式

Skill 在进行版本兼容性判断时，可以采用如下逻辑：

1. **识别用户当前组合** `(CANN_u, torch_u, torch_npu_u)`：
   - 若用户给出了确切版本：直接与矩阵对比；
   - 若未给出：仅能做“官方推荐组合”的一般性说明，标记为“当前环境版本未知”。
2. **查找是否存在完全匹配的组合**：
   - 若 `(CANN_u, torch_u, torch_npu_u)` 刚好与某一组合（如 A2）一致：
     - Skill 可以直接引用“该组合下算子支持度”的 reference 结论；
     - 在回答时说明：“在官方推荐组合 A2 下，此算子支持……”
3. **若无完全匹配，但部分版本相同**：
   - 例如用户为 `CANN=7.0.x, torch=2.3.0, torch_npu=2.1.0`：
     - 说明其与 A1 / A2 均不一致；
     - 指出可能风险：
       - 算子实现来自 `torch_npu 2.1.0`，但用户使用 `torch 2.3.0`，存在 ABI / API 差异；
       - 某些算子虽然在 `torch_npu 2.3.0` 中被标记为可用，但在 2.1.0 中可能尚不支持。
4. **若 CANN 版本低于推荐版本**：
   - 例如用户为 `CANN 6.x + torch 2.3.0 + torch_npu 2.3.0`：
     - 提醒：即便 torch_npu 与 PyTorch 版本匹配，CANN 过低仍可能缺少底层图算子或性能优化；
     - 对于某些数值/矩阵类算子，可能表现为：
       - `RuntimeError: op not found` / `function not implemented`；
       - 性能显著低于预期。

### 3. 描述推荐组合与风险的示例句式

- **推荐组合描述**：
  - “官方推荐使用 **CANN 8.0.RC1 + torch 2.7.1 + torch_npu 2.7.1** 这一组合，在该组合下，包含 softmax 在内的大部分训练常用算子都已经过适配和验证。”
- **不匹配风险描述**：
  - “你当前的组合为 **CANN 7.0.x + torch 2.3.0 + torch_npu 2.1.0**，与官方推荐组合 A1/A2 都不一致，可能导致部分算子在 NPU 上缺失或行为与文档不符（例如 softmax 只在 CPU 路径可用）。”
- **建议动作**：
  - “建议优先对齐到官方推荐的配套组合；若暂时无法升级，可先使用本 Skill 生成单算子脚本，在当前环境中实际运行 softmax/目标算子，以验证其具体支持情况和数值特性。”

### 4. 扩展建议

在实际项目中，可以进一步扩展：

- 为每行组合添加：
  - 支持的 PyTorch 大版本范围；
  - 支持的 Python 版本；
  - 推荐的 GCC / glibc 版本。
- 添加“是否适配动态图/torch.compile/Inductor”等特性；
- 添加“是否包含特定自定义算子集（如 transformer/moe/attention 相关）”的标记。

Skill 在使用本矩阵时，应避免将“精确版本选择”完全交给用户，而是：

- 给出 **1–2 个可行的推荐组合**；
- 结合用户约束（如“只能使用 PyTorch 2.1.0”）给出备选方案或风险提示。
